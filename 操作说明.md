## 五、技术架构

### 整体流程

```
视频帧 → YOLOv8s检测person → ByteTrack追踪 → 人物crop →
color_classifier判红/黄 → 滑动投票(12帧7票) → 角色锁定(8帧确认/15帧解锁) →
离岗检测(10s超时) → JSONL输出 + 标注视频
```

### 颜色分类器 v2.6.2

**红色检测 (严格)**:
- HSV: H=0-8°+172-180°, S>80, V>70
- BGR: R>G×1.25 且 R>B×1.4
- 目的: 排除棕色手套/橙色背景/旧黄设备

**黄色检测**: HSV H=15-55°, S>45, V>45

**中心加权**: 高斯椭圆权重, 中心1.0→边缘0.10

**过曝屏蔽**: V≥245 像素不参与计数

**多ROI** (4个): 中心躯干/上胸/左胸/右胸, 选黄ratio最高

**行/格覆盖**: 用密度阈值0.15判断颜色是"底色"还是"装饰(反光条)"

**判色优先级**:
1. 红底 + 黄装饰(低行覆盖) → 红
2. 黄底(高行覆盖) → 黄
3. 红强 → 红
4. 黄足够 → 黄
5. 否则 None

## 六、已知问题 (后续修复)

### 问题1: 红色背心+黄色反光条/文字 → 误判黄

测试1中, 红色监火背心(带黄色反光条)被判为黄背心。

**根因**: 严格红色HSV+BGR过滤在低分辨率下把真红背心也过滤掉了(red_ratio=0.0), 导致所有"红底优先"规则无数据可用。

**建议修复**:
1. 在 `_score_roi()` 加入宽松红色通道 (H=0-12, S>40, V>40, 不BGR过滤), 返回 `red_lenient_ratio`
2. 宽松红仅用于"底色存在性判断", 严格红用于确认
3. 判色: 当strict_red≈0但lenient_red>0.05且黄存在时, 计算ROI非黄区域平均R/(R+G+B), >0.38则判红底
4. ⚠️ 不要直接放宽严格红阈值 (会引入棕/橙误判)
5. 改后先跑测试2验证不退化, 再跑测试1验证修复

### 问题2: 低分辨率视频检出率低

704×576 ~ 960×544下YOLO检测和颜色分离度均下降, 属物理限制。

### 问题3: 泛化需调参

不同光照/摄像头需调整HSV阈值和黄红比例。新视频先跑看JSONL分布再调。

## 八、开发历史 (避免重复尝试)

| 版本 | 改动 | 结果 |
|------|------|------|
| v2.0 | YOLO-World v2 + RGB颜色 | 模型无法转RKNN, 放弃 |
| v2.3 | HSV-only, yellow优先, H=15-55 | 测试2黄色23% |

| v2.5 | 收紧红HSV+BGR+全姿态多ROI+黄连通块 | 测试2零切换, 测试1红底黄条判黄 |
| v2.6 | 反光条过滤(row_cov<0.18) | 维持测试2 |
| v2.6.1 | 增强条带检测 | 同v2.6 |
| v2.6.2 | 行/格覆盖区分底色vs装饰 | 当前版本 |
| v2.6.3 | 红色外框密度(outer_density) | 失败, 黄背心也判红, **已回退** |
