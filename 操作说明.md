## 三、模型权重说明

当前主流程在 `full_pipeline.py` 顶部写死默认权重：

```python
YOLO_WEIGHTS = str(ROOT / "models" / "yolov8s.pt")
```

因此运行前必须保证本地存在：

```text
models/yolov8s.pt
```

当前模型选择结论：

- `yolov8s.pt`：当前默认，综合检测率和速度最好。
- `yolov8n.pt`：速度快，但低分辨率场景检测率偏低。
- `yolov8l.pt`：更大，但当前测试中收益不明显，速度更慢。
- `yolov8l-worldv2_vest_finetuned.pt`：历史尝试权重，不是当前主流程默认。

注意：

- `models/*.pt` 已被 `.gitignore` 忽略，不进入 GitHub。
- 如果 clone 仓库后没有放权重，运行会报 `FileNotFoundError`。
- 权重建议通过网盘、服务器、移动硬盘或 GitHub Release 单独归档。

| `outputs/{视频名}_result.json` | 摘要报告，含总帧数、fps、告警次数、告警/清除事件 | 是 |

## 七、关键配置

`full_pipeline.py` 顶部关键参数：

| 参数 | 当前值 | 说明 |
| --- | --- | --- |
| `YOLO_WEIGHTS` | `models/yolov8s.pt` | 当前默认模型权重 |
| `CONF_THRES` | `0.45` | person 检测置信度 |
| `LEAVE_TIMEOUT` | `10.0s` | 连续无红色监火员超过该时间触发离岗 |
| `VOTE_WINDOW` | `12` | 颜色滑动投票窗口 |
| `VOTE_THRES` | `7` | 颜色确认票数阈值 |
| `LOCK_CONSEC` | `8` | 连续同色后锁定角色 |
| `UNLOCK_CONSEC` | `15` | 锁定后连续反色才解锁 |
| `SWITCH_MARGIN` | `0.08` | 解锁需要分数优势 |
| `MIN_BOX_W / MIN_BOX_H` | `50 / 100` | 最小 person 框尺寸 |

`leave_detector.py` 当前连续红色确认：

| 参数 | 当前值 | 说明 |
| --- | --- | --- |
| `present_confirm_s` | `0.3s` | 连续检测到红色达到该时长才刷新在岗时间 |
| `clear_confirm_s` | `0.5s` | 告警后连续检测到红色达到该时长才清除告警 |
